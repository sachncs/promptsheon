# ADR Template

Copy this file to `NNNN-short-slug.md` and fill in every section. Do not leave placeholders.

```markdown
# ADR NNNN: <Title in imperative mood>

- **Status:** Proposed | Accepted | Superseded | Deprecated
- **Date:** YYYY-MM-DD
- **Supersedes:** NNNN (if any)
- **Superseded by:** NNNN (if any)

## Context

What is the situation? What problem are we solving? What are the constraints
(technical, organisational, time, expertise)?

## Decision

What did we choose? State it in one or two sentences. The rest of the document
justifies it.

## Options considered

1. **<Name> (chosen).** Why this option.
2. **<Alternative>.** Why we did not choose it.
3. **<Alternative>.** Why we did not choose it.

## Consequences

What becomes easier? What becomes harder? What new failure modes appear? What
new responsibilities do we have?

## References

- `path/to/code.ext` — short description
- `docs/some-page.md` — short description
```
