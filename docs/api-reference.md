# API Reference

The full surface is generated from the route registrations in
`internal/api/server.go` and the handler signatures in
`internal/api/handlers_*.go`. The OpenAPI spec lives at
[`api/openapi.yaml`](../api/openapi.yaml) and is regenerated by
`go run ./scripts/genopenapi` (or `make openapi`).

This page is a curated, opinionated index of the endpoints an
operator or SDK consumer needs day-to-day. It is **not** a
replacement for the OpenAPI spec — it skips the trivial list
endpoints, focuses on lifecycle and integration surfaces, and
notes permissions and idempotency contracts.

## Conventions

- All endpoints are under `/api/v1/`.
- Authentication is `Authorization: Bearer <api_key>`. Most
  read endpoints require `PermPromptRead`; mutations require
  `PermPromptCreate` / `Update` / `Delete`; admin endpoints
  require `PermAuditRead` (audit + metrics + chain verify).
- IDs are server-generated strings. The OpenAPI spec documents
  the format; callers should treat them as opaque.
- Time fields are RFC 3339 UTC.
- All 2xx responses include a JSON body unless documented as
  empty (e.g. `204 No Content` on delete).

## Pagination

`GET` list endpoints accept `?limit=&offset=` query parameters.
Default `limit=50`, max `1000`; `limit<1` or `limit>1000`
returns 400. Successful responses include an `X-Total-Count`
header and an RFC 5988 `Link` header advertising `prev`,
`first`, `next`, and `last` pages.

## Idempotency

`POST` endpoints honour the `Idempotency-Key` request header.
The first response is cached in SQLite (migration 013) for
5 minutes; subsequent requests with the same `(method, path,
key, body-hash)` receive the cached response with an
`Idempotent-Replayed: true` header. Replicas share the cache
via SQLite, so a retried POST on a different daemon still
returns the original response.
